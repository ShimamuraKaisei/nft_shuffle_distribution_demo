# NFT シャッフル配布デモ — ビジネス概要

## 1. プロダクト概要

NFT をランダムに配布する体験を提供するフロントエンドデモアプリケーション。ユーザーが「シャッフル」ボタンを押すと、演出アニメーションとともにランダムに 1 つの NFT が当選する、ガチャ型の配布 UI を実装している。

**現在のステータス:** UIプロトタイプ v0.3（フロントエンドのみ、バックエンド未接続）

## 2. 解決する課題

| 従来の課題 | 本アプリによる解決 |
|---|---|
| NFT 配布が単調なリスト選択 | シャッフル演出によるエンタメ性の付与 |
| ユーザーの期待感が薄い | 段階的アニメーションでワクワク感を演出 |
| 配布フローの法務対応が未整備 | 利用規約同意フローの組み込み |
| 結果の拡散手段がない | X（Twitter）シェアボタンによる自然な拡散導線 |

## 3. 実装済みデモバリエーション

### Demo 1: ラインナップ表示あり（スタンダード）

- 配布対象の NFT 一覧と当選確率を事前に開示
- 透明性の高い運用に適したモデル
- 想定用途: コンプライアンス重視のキャンペーン、景表法対応が必要なケース

### Demo 2: ラインナップ非表示（ミステリーボックス型）

- 何が当たるか事前に見せない「お楽しみ」方式
- 3D 回転・パーティクルなど高品質なアニメーション演出
- 想定用途: サプライズ性を重視するマーケティング施策、エンゲージメント最大化

### Demo 3: 利用規約同意フロー付き

- シャッフル実行前に利用規約への同意を必須化
- 同意チェックボックスが OFF の間はボタンを無効化
- 想定用途: 法務要件を満たす必要がある本番運用

## 4. NFT ラインナップ（デモデータ）

| NFT 名 | レアリティ | 当選確率 |
|---|---|---|
| Dragon of Eternity | SSR | 1% |
| Phoenix Rising | SR | 9% |
| Mystic Wolf | R | 20% |
| Golden Eagle | R | 30% |
| Spirit Orb | N | 40% |

※ 実運用時はバックエンド API から動的に取得する想定。

## 5. ユーザーフロー

```
[キャンペーンページ訪問]
        ↓
[ラインナップ確認]（Demo1の場合）
        ↓
[利用規約に同意]（Demo3の場合）
        ↓
[シャッフルボタン押下]
        ↓
[アニメーション演出（約3秒）]
        ↓
[当選NFT発表 + コンフェッティ演出]
        ↓
[Xでシェア / 外部リンクへ遷移]
```

## 6. 技術スタック

| カテゴリ | 技術 |
|---|---|
| フレームワーク | React 19 + TypeScript |
| アニメーション | Framer Motion / CSS Animations |
| ビルドツール | Vite |
| UI設計 | モバイルファースト（レスポンシブ対応） |

## 7. 本番運用に向けて必要な追加開発

### 必須（MVP）

| 項目 | 概要 |
|---|---|
| バックエンド API | NFT ラインナップ取得・当選結果の確定をサーバーサイドで実行（不正防止） |
| ウォレット連携 | MetaMask 等のウォレット接続による NFT 受け取り |
| 認証・認可 | ユーザー認証と配布対象者の管理 |
| 残数管理 | リアルタイムの在庫管理と上限制御 |

### 推奨（品質向上）

| 項目 | 概要 |
|---|---|
| アクセシビリティ対応 | ARIA 属性の追加、スクリーンリーダー対応 |
| アナリティクス | シャッフル実行率・シェア率などの計測 |
| ダークモード | ユーザー設定に応じたテーマ切り替え |
| E2E テスト | Cypress / Playwright による自動テスト |
| 多言語対応 | i18n による日英対応 |

## 8. 想定される活用シーン

- **NFT プロジェクトのミント施策**: ランダム配布でコレクション価値を演出
- **企業キャンペーン**: デジタルグッズのガチャ形式配布
- **イベント来場特典**: 会場限定 NFT のエンタメ性ある配布
- **コミュニティリワード**: 貢献者へのサプライズ報酬

## 9. ライセンス・権利関係

- デモ用 NFT 画像は Arweave 上にホストされた外部アセットを使用
- 実運用時はオリジナルアセットの用意が必要
